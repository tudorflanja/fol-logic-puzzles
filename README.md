## üõ†Ô∏è Application Overview

This project contains **four classical logic puzzles**, each represented by a set of First-Order Logic constraints.  
The FOL files (e.g., `placings`, `firstLastNames`, `books`, `toddlersAndTeddies`) include:

- domain definitions  
- uniqueness constraints  
- logical predicates (e.g., `next(x,y)`, name‚Äìseries associations, ages, teddies)  
- clue translation into logic rules  
- Prover9/Mace4 compatible syntax  

### üîπ Included Puzzles

#### 1. **Placings**  
A race ordering puzzle with ten constraints.  
Uses **Prover9** due to the large model size.

#### 2. **First and Last Names**  
Match five people to their corresponding surnames using clues about hiking, children, gender, and hospital visits.

#### 3. **Toddlers and Teddies**  
Determine each child's age and teddy count using constraints such as:
- unique teddy counts  
- relative ages  
- exactly one matching age‚Äìteddy pair  

#### 4. **Books, Books and More Books**  
Assign each boy his favorite book series based on:
- preference for mysteries vs fantasy  
- character names  
- costume clues (e.g., glasses and black robe ‚Üí Harry Potter)

## üíª How the GUI Works

The GUI provides an accessible interface for navigating puzzles and revealing answers:

1. Displays four buttons‚Äîone per puzzle.  
2. For each puzzle, the GUI shows:
   - The puzzle description  
   - The logical clue list  
   - A **Show Answer** button that reveals the solution  
3. Built using Tkinter widgets:
   - `Label` for puzzle text  
   - `Button` for interaction  
4. Solutions are loaded from the project's puzzle model files.

## ‚ñ∂Ô∏è How to Run the Project

### 1. Install Dependencies
You only need Python and Tkinter:

sudo apt install python3 python3-tk

---

### 2. Run the GUI
Inside the logicsGUI folder:

python3 gui.py

---

### 3. Running the Puzzle Models
To generate or verify logic models manually:

prover9 -f placings
mace4 -f books
mace4 -f firstLastNames
mace4 -f toddlersAndTeddies

Note:
The Placings puzzle must be run with Prover9 because Mace4 cannot generate a model large enough.
